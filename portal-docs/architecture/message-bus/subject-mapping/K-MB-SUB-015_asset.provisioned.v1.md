# K-MB-SUB-015 — Asset Provisioned

> NATS subject mapping reference for new endpoint registration events within the Kubric UIDR platform.

## Subject Pattern

```
kubric.asset.provisioned.v1
kubric.asset.provisioned.>      # wildcard for all asset provisioned sub-subjects
```

Tokens:
| Position | Token           | Description                                  |
|----------|-----------------|----------------------------------------------|
| 1        | `kubric`        | Platform root namespace                      |
| 2        | `asset`         | Domain — asset management                    |
| 3        | `provisioned`   | Event type — new asset registration          |
| 4        | `v1`            | Schema version                               |

## Publisher

**Provisioning agent (Rust)** — the lightweight agent binary deployed to new endpoints during enrollment. The provisioning agent runs once during initial installation to register the asset with the Kubric UIDR platform.

Publish triggers:
| Trigger                 | Description                                               |
|-------------------------|-----------------------------------------------------------|
| First-time install      | Agent installed on a new endpoint for the first time      |
| Re-enrollment           | Agent re-registers after a factory reset or re-image      |
| Agent upgrade (major)   | Major version upgrade triggering re-registration          |

Install methods supported:
| Method        | Description                                |
|---------------|--------------------------------------------|
| `msi`         | Windows MSI installer via GPO or SCCM      |
| `deb`         | Debian/Ubuntu .deb package                 |
| `rpm`         | RHEL/CentOS .rpm package                  |
| `brew`        | macOS Homebrew cask                        |
| `docker`      | Containerized agent deployment             |
| `k8s_daemonset` | Kubernetes DaemonSet rollout            |

## Consumer(s)

| Consumer                 | Runtime              | Role                                                        |
|--------------------------|----------------------|--------------------------------------------------------------|
| **NOC** (primary)        | Go                   | Updates the central asset inventory, assigns monitoring policies |
| **K-SVC Portal** (primary) | TypeScript / Next.js | Displays new asset in tenant dashboard, triggers onboarding wizard |
| KAI-SENTINEL             | Python               | Recalculates health score with the new asset included        |
| Billing (KAI-CLERK)      | Python               | Increments licensed seat count for usage metering            |

## Payload

**Format:** JSON — Asset registration event with hardware fingerprint.

**Schema version:** `1.0.0`

**Content-Type:** `application/json`

```jsonc
{
  "schema_version": "1.0.0",
  "asset_id": "ast_r7s8t9u0",
  "tenant_id": "tnt_a1b2c3d4",
  "hostname": "WORKSTATION-087",
  "os": "Windows",
  "os_version": "11 Pro 23H2",
  "agent_version": "1.4.2",
  "ip_addresses": ["10.0.5.87", "fe80::1a2b:3c4d:5e6f:7890"],
  "mac_addresses": ["AA:BB:CC:DD:EE:01"],
  "fingerprint_blake3": "f8e7d6c5b4a3f8e7d6c5b4a3f8e7d6c5b4a3f8e7d6c5b4a3f8e7d6c5b4a3f8e7",
  "install_method": "msi",
  "enrolled_at": "2026-02-26T14:35:00Z",
  "hardware": {
    "cpu": "Intel Core i7-13700",
    "ram_gb": 32,
    "disk_gb": 512,
    "tpm_version": "2.0"
  },
  "tags": ["finance-dept", "pci-scope"]
}
```

## Fields

| Field                  | Type       | Required | Description                                                           |
|------------------------|------------|----------|-----------------------------------------------------------------------|
| `schema_version`       | string     | yes      | Payload schema version (semver)                                       |
| `asset_id`             | string     | yes      | Unique asset identifier (`ast_` prefix), generated by provisioning agent |
| `tenant_id`            | string     | yes      | Tenant the asset belongs to (`tnt_` prefix)                           |
| `hostname`             | string     | yes      | System hostname at time of enrollment                                 |
| `os`                   | string     | yes      | Operating system family: `Windows`, `Linux`, `macOS`, `Container`     |
| `os_version`           | string     | yes      | Full OS version string                                                |
| `agent_version`        | string     | yes      | Installed Kubric agent version (semver)                               |
| `ip_addresses`         | string[]   | yes      | All IP addresses (IPv4 and IPv6) assigned to the endpoint             |
| `mac_addresses`        | string[]   | yes      | All MAC addresses of network interfaces                               |
| `fingerprint_blake3`   | string     | yes      | BLAKE3 hash of hardware identifiers (CPU ID + disk serial + TPM EK) for unique device fingerprinting |
| `install_method`       | string     | yes      | Deployment method: `msi`, `deb`, `rpm`, `brew`, `docker`, `k8s_daemonset` |
| `enrolled_at`          | datetime   | yes      | ISO 8601 UTC timestamp of enrollment completion                       |
| `hardware.cpu`         | string     | no       | CPU model identifier                                                  |
| `hardware.ram_gb`      | integer    | no       | Total RAM in gigabytes                                                |
| `hardware.disk_gb`     | integer    | no       | Primary disk capacity in gigabytes                                    |
| `hardware.tpm_version` | string     | no       | TPM module version (if present)                                       |
| `tags`                 | string[]   | no       | Organizational tags for policy assignment (e.g., department, compliance scope) |

### Enrollment Workflow

The end-to-end enrollment process follows these steps:

```
1. Agent binary deployed via install_method
     |
2. Agent generates asset_id and computes fingerprint_blake3
     |
3. Agent authenticates to Kubric control plane via enrollment token
     |
4. Agent publishes kubric.asset.provisioned.v1
     |
5. NOC validates enrollment:
     ├── Check tenant license capacity
     ├── Verify fingerprint uniqueness (detect re-enrollment vs new)
     ├── Assign default monitoring policy based on os + tags
     └── Publish kubric.asset.policy.assigned.v1
     |
6. K-SVC Portal shows new asset in tenant dashboard
     |
7. KAI-CLERK increments licensed seat count
     |
8. KAI-SENTINEL includes asset in next health score calculation
```

### License Validation

On receiving a provisioned event, NOC checks the tenant's license allocation:

| Check                   | Action on Failure                                      |
|-------------------------|--------------------------------------------------------|
| Seat count within limit | Proceed with enrollment                                |
| Seat count exceeded     | Publish `kubric.asset.license.exceeded.v1` alert, enroll in degraded mode |
| Duplicate fingerprint   | Treat as re-enrollment, update existing asset record   |
| Invalid enrollment token | Reject enrollment, publish `kubric.asset.rejected.v1` |

**Subscribe to all asset provisioned events:**

```bash
nats sub "kubric.asset.provisioned.>"
```

**Create the JetStream stream:**

```bash
nats stream add ASSET_PROVISIONED \
  --subjects "kubric.asset.provisioned.>" \
  --storage file \
  --retention limits \
  --max-age 365d \
  --max-bytes 5GB \
  --replicas 3 \
  --discard old \
  --dupe-window 60s
```

**Create the noc-inventory consumer:**

```bash
nats consumer add ASSET_PROVISIONED noc-inventory \
  --deliver all \
  --ack explicit \
  --max-deliver 5 \
  --filter "kubric.asset.provisioned.>"
```

## Notes

- **Fingerprint Uniqueness:** The `fingerprint_blake3` hash ensures that hardware re-images or OS reinstalls on the same physical device are detected as re-enrollments rather than new assets, preventing license count inflation.
- **Retention Policy:** Asset provisioned events are retained for 365 days (1 year) in JetStream for audit and compliance purposes. This supports SOC 2 asset inventory requirements.
- **Re-enrollment Handling:** When a duplicate `fingerprint_blake3` is detected, NOC updates the existing asset record with the new `agent_version`, `ip_addresses`, and `enrolled_at` timestamp rather than creating a duplicate.
- **Tag-Based Policy Assignment:** The `tags` field drives automatic policy assignment. For example, assets tagged `pci-scope` receive the PCI-DSS compliance monitoring policy, while `finance-dept` assets receive enhanced FIM monitoring on financial data directories.
- **Agent Heartbeat:** After provisioning, the agent begins publishing heartbeats to `kubric.asset.heartbeat.v1` at 60-second intervals. An asset with no heartbeat for 5 minutes is flagged as offline in the NOC dashboard.
- **Related Subjects:** `kubric.asset.heartbeat.v1` (ongoing agent health), `kubric.asset.policy.assigned.v1` (monitoring policy assignment), `kubric.asset.decommissioned.v1` (asset removal), `kubric.asset.license.exceeded.v1` (over-limit alert).
