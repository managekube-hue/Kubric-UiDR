# K-MB-SUB-013 — security.alert.v1

---

## 1. Overview

| Field              | Value                                                           |
|--------------------|------------------------------------------------------------------|
| **Subject**        | `kubric.<tenant_id>.security.alert.v1`                          |
| **Stream**         | `SECURITY-ALERT`                                                |
| **Version**        | v1                                                              |
| **Purpose**        | Central security event bus for the Kubric platform. Carries every detection-quality alert generated by the coresec Rust detection engine (Sigma and YARA rule matches), the behavioural anomaly model, the threat intelligence correlation pipeline, and the CrewAI triage agent. Each message is a structured alert record with full evidence chain, MITRE ATT&CK mapping, asset context, and lifecycle disposition tracking. Primary feed for analyst workflow, incident management, ML training, and SIEM forwarding. |
| **Owner**          | Security Operations / Detection Engineering                     |
| **SLA**            | At-least-once delivery; critical/high alerts must be persisted by `analyst-agent` within 10 seconds; 365-day retention for legal hold compliance |

---

## 2. NATS Configuration (YAML)

```yaml
stream:
  name: SECURITY-ALERT
  subjects:
    - "kubric.*.security.alert.v1"
  retention: limits
  max_age: 31536000        # 365 days — one full year for compliance
  max_bytes: 107374182400  # 100 GB
  replicas: 3
  storage: file
  discard: old
  duplicate_window: 120s

consumer:
  name: analyst-agent
  durable: analyst-agent
  ack_policy: explicit
  max_deliver: 5
  ack_wait: 30s
  deliver_policy: all
```

> Additional durable consumers `hunter-agent`, `incident-manager`, `siem-forwarder`, and `ml-pipeline` share this stream with identical ack settings.

## 5. Publisher Details

| Attribute          | Value                                                                                                          |
|--------------------|----------------------------------------------------------------------------------------------------------------|
| **Service A**      | coresec Rust detection engine (Sigma/YARA)                                                                     |
| **Service B**      | CrewAI triage agent                                                                                            |
| **Trigger A**      | Every Sigma or YARA rule match detected by the coresec engine reading from the EDR/NDR/ITDR event streams      |
| **Trigger B**      | CrewAI triage agent publishes enriched alerts after correlation with TI-IOC stream and CMDB asset context      |
| **Subject format** | `kubric.{tenant_id}.security.alert.v1`                                                                         |
| **Library**        | Rust: `async-nats` crate JetStream `publish_with_headers`; Python: `nats.py` JetStream `publish_async`        |
| **Nats-Msg-Id**    | `{tenant_id}:{alert_id}` prevents duplicate alert publication during detection engine restart                  |
| **Serialisation**  | UTF-8 JSON                                                                                                     |
| **Headers**        | `Kubric-Tenant`, `Kubric-Schema-Version: v1`, `Kubric-Severity: {severity}`, `Kubric-Alert-Type: {type}`      |

## 6. Consumer Details

### analyst-agent
| Attribute          | Value                                                                                             |
|--------------------|---------------------------------------------------------------------------------------------------|
| **Role**           | Receives alerts and enriches them with CMDB asset context and recent TI-IOC matches. Writes a structured triage report to Postgres `soc.alerts` table. Assigns an analyst queue entry for critical/high severity within 10 seconds of receipt. |
| **Batch size**     | 10 messages per pull (low latency prioritised)                                                    |
| **Error behaviour**| NACKs on Postgres write failure; dead-letters after 5 redeliveries                               |

### hunter-agent
| Attribute          | Value                                                                                             |
|--------------------|---------------------------------------------------------------------------------------------------|
| **Role**           | On `sigma_match` or `yara_match` type, launches a retrohunt across the 30-day telemetry window in ClickHouse to identify additional affected assets. Publishes new alerts for any additional rule matches found. |
| **Batch size**     | 5 messages per pull (each triggers expensive ClickHouse queries)                                  |
| **Filter**         | Only processes `severity IN (critical, high)` to control ClickHouse query volume                  |
| **Error behaviour**| NACKs with 30 s delay on ClickHouse timeout; dead-letters after 5 redeliveries                   |

### incident-manager
| Attribute          | Value                                                                                             |
|--------------------|---------------------------------------------------------------------------------------------------|
| **Role**           | Correlates related alerts by asset, technique, and time window into incidents. Creates or updates Postgres `soc.incidents` records. Triggers the Temporal workflow `incident-response-wf` for critical alerts. |
| **Batch size**     | 50 messages per pull                                                                              |
| **Error behaviour**| NACKs on Temporal workflow start failure; dead-letters after 5 redeliveries                      |

### siem-forwarder
| Attribute          | Value                                                                                             |
|--------------------|---------------------------------------------------------------------------------------------------|
| **Role**           | Translates alerts to CEF format and forwards to the tenant SIEM endpoint via syslog-TLS or REST. Applies TLP handling for `threat_intel` type alerts per redistribution rules. |
| **Batch size**     | 200 messages per pull                                                                             |
| **Error behaviour**| NACKs with 15 s delay on SIEM unreachable; dead-letters after 5 redeliveries                     |

### ml-pipeline
| Attribute          | Value                                                                                             |
|--------------------|---------------------------------------------------------------------------------------------------|
| **Role**           | Streams alert features into the online feature store (Redis) for real-time anomaly model inference and into S3 as Parquet for offline model retraining. Tracks label changes via the `disposition` field as training signal. |
| **Batch size**     | 500 messages per pull                                                                             |
| **Error behaviour**| NACKs on S3 write failure; dead-letters after 5 redeliveries                                     |

## 7. Retention Policy

| Parameter            | Value                                                                                          |
|----------------------|------------------------------------------------------------------------------------------------|
| **Stream retention** | `limits`                                                                                       |
| **max_age**          | 31,536,000 seconds (365 days) — legal hold compliance requirement                              |
| **max_bytes**        | 100 GB                                                                                         |
| **Replicas**         | 3                                                                                              |
| **Storage tier**     | `file`                                                                                         |
| **Archive policy**   | ClickHouse `security.alert_events` retains all alerts indefinitely. ClickHouse TTL tiers rows older than 2 years to S3 Parquet. S3 objects are subject to a 7-year immutable retention policy via S3 Object Lock for compliance. |

## 8. Error Handling

| Scenario                              | Behaviour                                                                                                                              |
|---------------------------------------|----------------------------------------------------------------------------------------------------------------------------------------|
| **Detection engine restart**          | `Nats-Msg-Id: {tenant_id}:{alert_id}` prevents duplicate publication within the 120-second window. The coresec engine uses a local SQLite checkpoint to avoid reprocessing events already emitted. |
| **Critical alert consumer lag**       | A lag monitor checks `analyst-agent` consumer lag every 30 seconds. If lag > 50 messages on critical severity, Prometheus fires `security_alert_consumer_lag_high` and PagerDuty pages the on-call detection engineer. |
| **Schema validation failure**         | Invalid messages forwarded immediately to `kubric.<tenant_id>.dlq.security.alert.v1` with `Kubric-Error-Reason: SCHEMA_INVALID`. |
| **Max redeliveries exceeded**         | NATS routes to `kubric.<tenant_id>.dlq.security.alert.v1`. PagerDuty alert at DLQ depth > 5 for critical severity. |
| **Alert storm (>1000 alerts/min)**    | incident-manager activates storm mode: deduplicates by `(asset_id, mitre_technique)` within a 15-minute window, grouping correlated detections into a single incident. |
| **SIEM endpoint unreachable**         | siem-forwarder queues messages locally up to 10,000 and retries with exponential backoff. If buffer fills, drops `info` severity to protect `critical` delivery throughput. |
| **Dead-letter subject**               | `kubric.<tenant_id>.dlq.security.alert.v1`                                                                                             |
