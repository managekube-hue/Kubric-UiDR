# K-HV-002 — Ceph Distributed Storage on Proxmox

> **Cluster:** 3-node Ceph cluster on pve-kubric-01/02/03  
> **OSD Drives:** NVMe from DATA_RAID5 partitions  
> **Pools:** kubric-vms (replicated ×3), kubric-data (replicated ×2)  
> **CephFS:** kubric-shared (for config/shared state)  
> **Network:** Ceph cluster traffic on 10.0.200.0/24 (vmbr1, MTU 9000)

---

## 1. Prerequisites

```bash
# On ALL nodes — verify storage network
ping -c 3 -s 8972 10.0.200.22   # Jumbo frame test
ip addr show vmbr1 | grep "inet "

# Verify NVMe partitions available for OSD
lsblk -o NAME,SIZE,TYPE,MOUNTPOINT | grep -v "loop\|sr0"
# Look for raw partitions (no filesystem, no mount)
```

## 3. Ceph Configuration

### 3.1 /etc/pve/ceph.conf

```ini
[global]
    fsid = <auto-generated UUID>
    auth_cluster_required = cephx
    auth_service_required = cephx
    auth_client_required = cephx

    # Network
    public_network = 10.0.200.0/24
    cluster_network = 10.0.200.0/24

    # Performance — NVMe optimized
    osd_pool_default_size = 3
    osd_pool_default_min_size = 2
    osd_pool_default_pg_num = 128

    # OSD settings for NVMe
    osd_memory_target = 4294967296       # 4 GB per OSD
    osd_op_threads = 8
    osd_op_num_shards = 8

    # BlueStore settings (NVMe tuning)
    bluestore_cache_size_ssd = 4294967296   # 4 GB
    bluestore_min_alloc_size_ssd = 4096
    bluestore_block_size = 0               # use entire device

    # Recovery tuning
    osd_recovery_max_active = 3
    osd_recovery_op_priority = 3
    osd_max_backfills = 2

    # Scrub scheduling (off-peak)
    osd_scrub_begin_hour = 2
    osd_scrub_end_hour = 6

[mon]
    mon_allow_pool_delete = true
    mon_warn_on_pool_no_redundancy = true

[mgr]
    mgr_modules = dashboard,prometheus
```

## 5. OSD Creation

### 5.1 Identify OSD Devices

```bash
# On each node — find the raw Ceph partition
lsblk -o NAME,SIZE,TYPE,FSTYPE,MOUNTPOINT

# Node 1: /dev/sdb (or ceph partition from K-HW-001)
# Node 2: /dev/sdb3 (~900GB partition from K-HW-002)
# Node 3: /dev/sdb2 (~1.8TB partition from K-HW-003)

# Wipe any existing data
ceph-volume lvm zap /dev/sdb --destroy
```

### 5.2 Create OSDs

```bash
# On pve-kubric-01
pveceph osd create /dev/sdb
# Or if using a partition:
# pveceph osd create /dev/sdb3

# On pve-kubric-02
pveceph osd create /dev/sdb3

# On pve-kubric-03
pveceph osd create /dev/sdb2

# Verify OSDs
ceph osd tree
# Expected:
# ID  CLASS  WEIGHT   TYPE NAME              STATUS  REWEIGHT  PRI-AFF
# -1         X.XXXXX  root default
# -3         X.XXXXX      host pve-kubric-01
#  0    ssd  X.XXXXX          osd.0              up   1.00000  1.00000
# -5         X.XXXXX      host pve-kubric-02
#  1    ssd  X.XXXXX          osd.1              up   1.00000  1.00000
# -7         X.XXXXX      host pve-kubric-03
#  2    ssd  X.XXXXX          osd.2              up   1.00000  1.00000
```

### 5.3 Set OSD Class (NVMe)

```bash
# Mark all OSDs as SSD/NVMe class
ceph osd crush set-device-class ssd osd.0 osd.1 osd.2
```

## 6. Pool Creation

### 6.1 kubric-vms Pool (VM Images — Replicated ×3)

```bash
pveceph pool create kubric-vms \
  --pg_num 128 \
  --size 3 \
  --min_size 2 \
  --crush_rule replicated_rule \
  --application rbd

# Set autoscaler
ceph osd pool set kubric-vms pg_autoscale_mode on

# Verify
ceph osd pool ls detail | grep kubric-vms
```

### 6.2 kubric-data Pool (Data — Replicated ×2)

```bash
pveceph pool create kubric-data \
  --pg_num 64 \
  --size 2 \
  --min_size 1 \
  --crush_rule replicated_rule \
  --application rbd

ceph osd pool set kubric-data pg_autoscale_mode on
```

### 6.3 Register Pools in Proxmox Storage

```bash
# VM images pool
pvesm add rbd kubric-ceph \
  --pool kubric-vms \
  --monhost 10.0.200.21,10.0.200.22,10.0.200.23 \
  --content images,rootdir \
  --krbd 0

# Data pool (for containers, backups)
pvesm add rbd kubric-data-pool \
  --pool kubric-data \
  --monhost 10.0.200.21,10.0.200.22,10.0.200.23 \
  --content images,rootdir \
  --krbd 0

pvesm status | grep kubric
```

## 7. CephFS for Shared Storage

### 7.1 Create MDS (Metadata Server)

```bash
# Create MDS on at least 2 nodes for HA
pveceph mds create   # on node 1 (active)
# On node 2:
pveceph mds create   # standby

ceph mds stat
# Expected: kubric-shared:1 {0=pve-kubric-01=up:active} 1 up:standby
```

### 7.2 Create CephFS

```bash
# Create data and metadata pools
ceph osd pool create cephfs_data 64
ceph osd pool create cephfs_metadata 32

# Create filesystem
ceph fs new kubric-shared cephfs_metadata cephfs_data

# Mount on each node
mkdir -p /mnt/ceph-shared
cat >> /etc/fstab <<'EOF'
10.0.200.21,10.0.200.22,10.0.200.23:/ /mnt/ceph-shared ceph name=admin,secret=$(ceph auth get-key client.admin),noatime,_netdev 0 0
EOF
mount /mnt/ceph-shared

# Create directories for shared configs
mkdir -p /mnt/ceph-shared/{kubric-config,ssl-certs,detection-rules,model-cache}
```

### 7.3 Register CephFS in Proxmox

```bash
pvesm add cephfs kubric-cephfs \
  --monhost 10.0.200.21,10.0.200.22,10.0.200.23 \
  --content snippets,backup,iso \
  --fs-name kubric-shared
```

## 8. Performance Tuning for NVMe

### 8.1 I/O Scheduler

```bash
# Set none (noop) scheduler for NVMe — lowest latency
for dev in /sys/block/nvme*/queue/scheduler; do
  echo none > $dev
done

# Persistent via udev
cat > /etc/udev/rules.d/60-io-scheduler.rules <<'EOF'
ACTION=="add|change", KERNEL=="nvme*", ATTR{queue/scheduler}="none"
EOF
```

### 8.2 Read-Ahead Tuning

```bash
# Reduce read-ahead for random I/O workloads (ClickHouse, PostgreSQL)
for dev in /sys/block/nvme*/queue/read_ahead_kb; do
  echo 128 > $dev
done
```

### 8.3 Ceph OSD Tuning

```bash
# Per-OSD memory target (already in ceph.conf)
ceph config set osd osd_memory_target 4294967296

# BlueStore async compaction
ceph config set osd bluestore_compression_algorithm snappy
ceph config set osd bluestore_compression_mode aggressive

# Increase PG log size for NVMe speed
ceph config set osd osd_min_pg_log_entries 3000
ceph config set osd osd_max_pg_log_entries 10000
```

## 9. Recovery Procedures

### 9.1 OSD Failure Recovery

```bash
# Check cluster health
ceph health detail

# If OSD is down:
ceph osd tree | grep down
# Example: osd.1 is down

# Check which drive failed
ceph osd metadata osd.1 | grep bluefs_db_dev_node

# Replace drive and recreate OSD:
ceph osd out osd.1
ceph osd purge osd.1 --yes-i-really-mean-it

# After physical drive replacement:
pveceph osd create /dev/sdX
```

### 9.2 Monitor Recovery

```bash
# If a monitor is lost
ceph mon stat

# Remove dead monitor
ceph mon remove pve-kubric-XX

# Re-add on repaired node
pveceph mon create
```

### 9.3 Forced Recovery (Emergency)

```bash
# If cluster is in HEALTH_ERR with stuck PGs:
ceph pg dump_stuck unclean
ceph osd unset norecover
ceph osd unset nobackfill

# Force PG repair
ceph pg repair <pg_id>
```
